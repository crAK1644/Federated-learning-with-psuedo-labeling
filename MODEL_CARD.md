# Model Card

## Backbones (`src/ssfl/models.py`, paper Table I)

Every backbone maps its input to a 128-dim embedding; an 11-way classifier head and a 2-way
discriminator head (SSFL only) are each a single `Linear(128, out_dim)` on top of the same
embedding — a classifier and discriminator for a given backbone share every parameter except that
final head.

| Backbone | Input | Architecture |
|---|---|---|
| **CNN** (default) | `(batch, 23, 5)` — Eq. 19 reshape of the 115-feature vector | 8× `Conv1d` (conv1-4: 64ch k3 s1 p1; conv5-6: 128ch k3 s1 p1; conv7-8: 128ch k3 **s2** p1, length 5→3→2), ReLU each, flatten `(128,2)=256` → `Dense(256,128)` + ReLU |
| **MLP** | `(batch, 115)` flat | `115 → 512 → 256 → 128`, ReLU between layers |
| **LSTM** | `(batch, 23, 5)` transposed to `(batch, 5, 23)` | 2-layer LSTM, hidden=128, batch-first, over 5 timesteps × 23 features; final hidden state → `Dense(128,128)` + ReLU |

No batch norm, dropout, or softmax inside any backbone/head — cross-entropy is computed on raw
logits; softmax is applied only where the protocol explicitly needs probabilities (confidence
scores, DS-FL distillation targets, FD class-prob averaging).

## Training procedure (`src/ssfl/training.py`)

Adam, lr `1e-4`, 5 local epochs per phase, batch size 80 (paper prose says 100 — see
`REPRODUCIBILITY.md` for that discrepancy). Adam state is reinitialized every Flower task; model
weights persist across rounds in `Context.state`. All RNG (Python/NumPy/torch CPU+GPU/DataLoader
workers/partitioning/init) is seeded from a single run seed; `paper` profile uses deterministic
kernels on CUDA and allocates eight concurrent actors at 0.125 GPU each on one RTX 3090.

## Protocols (what differs per algorithm — see `SECURITY.md` for the verified privacy boundary)

- **SSFL** — classifier + discriminator per client; discriminator filters which open-set examples
  are "familiar" enough to pseudo-label; server majority-votes filtered labels across clients, then
  broadcasts consensus hard labels for a second distillation phase. This repo's primary target.
- **FL** — standard FedAvg baseline (sample-weighted average of uploaded classifier params).
- **FD** — clients upload per-class mean softmax vectors instead of params; server does leave-self-out
  class-wise averaging (Eq. 4); clients distill against the returned teacher targets.
- **DS-FL** — clients upload full per-example soft predictions over the open set; server averages and
  sharpens (`T=0.1`) before broadcasting distillation targets.

## Intended use

Research reproduction of the SSFL paper's IoT intrusion-detection setup (N-BaIoT, 11-class
benign/gafgyt/mirai classification) under three non-IID federated client scenarios. Not intended
for deployment against live network traffic — training and evaluation both use the same
mini-N-BaIoT static dataset (`DATA_CARD.md`), and no online/streaming inference path exists in this
repo.

## Evaluation

Accuracy, macro precision/recall/F1 (`zero_division=0`), per-class P/R/F1/support, confusion
matrices (normalized + raw), computed against the held-out test split every round. FL, SSFL, and
DS-FL use the server model. FD evaluates every personalized client on the same test split and uses
the highest-accuracy participating client, matching the paper's stated FD reporting convention.
Every batch/epoch/round and GPU sample is also retained in attempt-scoped JSONL telemetry.
Exact reproduced-vs-paper numbers, per scenario/algorithm/ablation, are in
`artifacts/report/` (generated by `ssfl.reporting.build_report`) and cross-referenced against the
paper's Table II/III/IV values in `REPRODUCIBILITY.md` — not duplicated here since those numbers
depend on which profile (`smoke`/`paper`) and how many seeds were actually run.

## Known limitations

- **CPU determinism**: two identical-seed CPU runs agree only within a small tolerance
  (`abs=0.05` on bounded metrics, `rel=0.05` on loss), not bit-exactly — root-caused to Apple
  Silicon's heterogeneous P/E-core floating-point behavior combined with Ray not pinning worker
  CPU affinity; amplified by DS-FL's `sharpen(T=0.1)`. See `REPRODUCIBILITY.md` #28.
- **Paper-scale (200-round, 27/89-client) runs are configured but not executed** in this repo — gated
  on GPU hardware; the `paper` profile in `configs/paper.yaml` is ready to run there.
- **No robustness to a colluding majority** of sampled clients in pseudo-label voting/averaging —
  by design, matches the paper's scope (see `SECURITY.md` threat model).
- FC layer width, DS-FL temperature interpretation, and FD teacher-weighting are all
  paper-ambiguous choices this repo had to pick; every one is logged with its reasoning in
  `REPRODUCIBILITY.md`.
